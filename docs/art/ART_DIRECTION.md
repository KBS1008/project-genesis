# Art Direction

**Project:** Project Genesis  
**Version:** 1.0  
**Status:** Planning  
**Last Updated:** 2026-07-12

---

# Purpose

This document defines the official visual identity of Project Genesis.

It serves as the single source of truth for all graphical assets and ensures a consistent artistic vision across the entire project.

Every illustration, icon, interface element, animation and promotional asset must follow these guidelines.

---

# Vision

Project Genesis is a modern economic simulation.

The visual style should communicate:

- Intelligence
- Precision
- Professionalism
- Stability
- Progress
- Industrial Growth
- Technological Evolution

The game should feel believable rather than cartoonish.

Players should immediately understand information without unnecessary visual distractions.

---

# Core Design Principles

Every asset should support the following principles.

## Readability First

Information always has priority over decoration.

Players will spend hundreds of hours interacting with dashboards, statistics and production chains.

Visual clarity is therefore more important than artistic complexity.

---

## Consistency

Every asset should look like it belongs to the same game.

The player should never notice that different AI tools or artists contributed to the project.

---

## Modular Design

Assets should be reusable.

Buildings, icons and UI components should share common proportions and visual language.

---

## Functional Beauty

Beauty should emerge from clean design and logical structure rather than visual effects.

Avoid unnecessary ornamentation.

---

## Scalability

Assets should remain usable:

- on small monitors
- on 4K displays
- in future UI redesigns

---

# Artistic Style

Project Genesis combines several influences.

Primary inspirations:

- modern management software
- engineering blueprints
- industrial architecture
- geographic information systems (GIS)
- contemporary city-building games

The overall appearance should be:

- clean
- realistic
- elegant
- timeless

---

# Visual Mood

Desired atmosphere:

- calm
- optimistic
- productive
- innovative
- industrial
- organized

Avoid:

- dystopian themes
- exaggerated fantasy
- post-apocalyptic visuals
- comic aesthetics

---

# Perspective

Preferred perspective:

- Isometric

Alternative views:

- Orthographic
- Top-down for maps

Avoid perspective distortion.

---

# Lighting

Lighting should be:

- natural
- soft
- neutral

Avoid:

- dramatic shadows
- excessive bloom
- high saturation
- cinematic lighting

The UI should remain visually separated from the world.

---

# Materials

Materials should feel authentic.

Examples:

- Steel
- Glass
- Concrete
- Wood
- Brick
- Water
- Asphalt
- Vegetation

Materials should be recognizable without excessive texture detail.

---

# Color Philosophy

Colors communicate information.

Players should quickly recognize:

- production
- finance
- logistics
- research
- energy

The official palette is defined in:

```
COLOR_PALETTE.md
```

---

# Shapes

Preferred shapes:

- rectangles
- rounded rectangles
- circles
- hexagons (technology)
- grids

Avoid unnecessary complexity.

---

# Line Style

Use:

- clean outlines
- subtle borders
- consistent line width

Avoid:

- sketch effects
- hand-drawn outlines
- comic strokes

---

# User Interface

The UI should resemble professional business software.

Characteristics:

- clean
- spacious
- predictable
- minimalistic

Animations should support interaction rather than distract from it.

---

# Icons

Icons should be:

- instantly recognizable
- simple
- monochrome-friendly
- scalable
- visually balanced

Icons should communicate meaning without text whenever possible.

---

# Buildings

Buildings should:

- represent their function
- have consistent proportions
- use realistic materials
- avoid exaggerated decoration

Industrial buildings should prioritize functionality.

---

# Resources

Every resource should have:

- one primary icon
- one simplified icon
- optional monochrome version

Icons should remain readable at small sizes.

---

# Maps

Maps should emphasize readability.

Layers include:

- terrain
- regions
- infrastructure
- buildings
- logistics
- overlays

Maps should avoid excessive visual noise.

---

# Charts

Charts should:

- use consistent colors
- minimize unnecessary gridlines
- emphasize trends
- avoid decorative effects

The goal is analytical clarity.

---

# Typography

Typography should be:

- modern
- neutral
- highly readable

Avoid decorative fonts.

Use a limited number of font families throughout the game.

---

# Animation Philosophy

Animations should communicate state changes.

Examples:

- production started
- construction completed
- research finished

Animations should be:

- short
- subtle
- meaningful

Avoid excessive motion.

---

# Accessibility

The visual language should support:

- high contrast
- color-blind accessibility
- scalable UI
- readable fonts
- icon redundancy

Color should never be the sole method of conveying important information.

---

# AI Asset Generation

All AI-generated assets should follow standardized prompts.

Prompts should define:

- style
- perspective
- lighting
- materials
- color palette
- level of detail
- background

No asset should be generated without referencing this document.

---

# Asset Review Checklist

Every new asset should answer:

- Does it match the Art Direction?
- Is it readable?
- Does it follow the color palette?
- Is the perspective correct?
- Is the lighting consistent?
- Does it fit existing assets?
- Can it be reused?
- Is it scalable?

Only approved assets become part of the official asset library.
