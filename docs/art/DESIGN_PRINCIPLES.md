# Purpose

This document defines the fundamental design principles that guide the creation of every visual asset in Project Genesis.

These principles translate the Visual Pillars into practical design rules that can be consistently applied by artists, designers and AI-assisted workflows.

Whenever multiple design options exist, these principles should guide the final decision.

# Principle 1 — Clarity Before Decoration

Visual elements exist to communicate information.

Decoration should never reduce readability.

Every pixel should contribute either to understanding or visual hierarchy.

Prefer:

- simple compositions
- clear silhouettes
- strong contrast
- balanced spacing

Avoid:

- excessive ornamentation
- unnecessary visual effects
- decorative complexity

---

# Principle 2 — Consistent Visual Hierarchy

Players should instantly recognize what is most important.

Visual hierarchy should be created through:

- size
- contrast
- spacing
- positioning
- color emphasis

Important information should never compete with secondary information.

---

# Principle 3 — Simplicity Through Reduction

Complex systems should appear approachable.

Whenever possible:

- remove unnecessary elements
- merge similar components
- simplify shapes
- reduce visual noise

Complexity belongs to the simulation—not to the presentation.

---

# Principle 4 — Functional Representation

Every visual object should communicate its purpose.

Buildings, resources and UI elements should be understandable without reading labels.

The player should recognize function through form.

---

# Principle 5 — Consistency

Consistency builds trust.

Every asset should follow the same rules regarding:

- proportions
- perspective
- lighting
- color usage
- spacing
- typography
- iconography

Consistency has higher priority than individual artistic expression.

---

# Principle 6 — Modular Construction

Visual assets should be designed as reusable building blocks.

Examples include:

- icons
- buttons
- windows
- buildings
- overlays

Modularity improves scalability and long-term maintainability.

---

# Principle 7 — Information Density with Breathing Room

Economic simulations naturally display large amounts of information.

The interface should therefore balance:

- dense information
- generous spacing
- clear grouping
- predictable layouts

Whitespace is an intentional design element.

---

# Principle 8 — Predictability

Users should quickly learn the interface.

Similar interactions should always produce similar visual responses.

Examples:

- identical button states
- consistent icon meanings
- uniform notification styles
- standardized dialogs

Predictability reduces cognitive load.

---

# Principle 9 — Accessibility

Accessibility is part of the design process.

Assets should support:

- high contrast
- scalable interfaces
- keyboard navigation
- color-blind users
- readable typography

Important information should never rely on color alone.

---

# Principle 10 — Timeless Design

Avoid visual trends that may quickly become outdated.

Prefer:

- clean geometry
- restrained effects
- balanced proportions
- professional color usage

The game should remain visually appealing for many years.

---

# Principle 11 — Performance Awareness

Visual quality should never compromise simulation performance.

Prefer:

- reusable assets
- optimized textures
- efficient icon systems
- lightweight UI elements

The design should scale to a wide range of hardware.

---

# Principle 12 — AI-Friendly Design

Project Genesis is designed for AI-assisted asset production.

Design decisions should therefore be:

- clearly describable
- repeatable
- measurable
- modular

Every asset should be reproducible using standardized prompts.

---

# Decision Framework

When evaluating multiple design options, apply the following order:

1. Readability
2. Functionality
3. Consistency
4. Accessibility
5. Reusability
6. Performance
7. Aesthetics

If aesthetics conflict with readability, readability always wins.

Before approving a new asset, verify:

| Question | Status |
|-----------|--------|

| Can it be reused? | □ |
