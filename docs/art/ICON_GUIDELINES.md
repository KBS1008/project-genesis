# Icon Guidelines

**Project:** Project Genesis  
**Version:** 1.0  
**Status:** Active  
**Last Updated:** 2026-07-12

---

# Purpose

This document defines the official icon system for Project Genesis.

Icons are one of the primary methods of communicating information throughout the game. They appear in the user interface, dashboards, production chains, maps, buildings, research trees and notifications.

A consistent icon language improves readability, reduces cognitive load and accelerates player decision-making.

# Icon Philosophy

Icons are visual language.

Every icon should answer one question:

> Can a player understand the meaning within one second?

If not, the icon is too complex.

---

# Design Goals

Icons should be:

- immediately recognizable
- simple
- consistent
- scalable
- timeless
- reusable
- accessible

---

# Design Style

Preferred style:

- Flat
- Minimalistic
- Geometric
- Clean
- Professional

Avoid:

- Cartoon style
- Fantasy style
- Comic style
- Excessive realism
- Decorative illustration

---

# Shape Language

Preferred geometry:

- circles
- rounded rectangles
- squares
- simple polygons

Technology-related icons may use hexagonal structures.

Complex silhouettes should be avoided.

---

# Line Style

Icons should use:

- consistent stroke width
- smooth corners
- clean geometry
- balanced proportions

Do not mix outline and filled styles within the same icon family.

---

# Filled vs Outline

Primary UI Icons

Outline

Status Indicators

Filled

Warnings

Filled

Notifications

Filled

Resource Icons

Filled

Toolbar Icons

Outline

The style should remain consistent throughout each category.

---

# Visual Complexity

Every icon should remain readable at small sizes.

Preferred characteristics:

- one primary symbol
- minimal internal details
- high silhouette recognition
- balanced spacing

Avoid unnecessary visual details.

---

# Perspective

Icons should use:

- front view
- orthographic projection
- simplified geometry

Avoid perspective distortion.

---

# Color Usage

Icons should primarily use neutral colors.

Color should communicate status rather than decoration.

Examples:

Blue

Interactive

Green

Positive

Orange

Warning

Red

Error

Gray

Neutral

Refer to COLOR_PALETTE.md for official values.

---

# Shadows

Icons generally should not use shadows.

If required:

- subtle
- consistent
- minimal opacity

Flat icons are preferred.

---

# Interactive States

Every interactive icon should define:

- Normal
- Hover
- Active
- Disabled

State changes should be subtle.

---

# Sizes

Recommended categories:

- Extra Small
- Small
- Medium
- Large
- Extra Large

Exact pixel sizes will later be defined in DESIGN_TOKENS.md.

---

# Consistency Rules

Every icon should maintain:

- identical stroke width
- identical corner radius
- consistent proportions
- consistent spacing
- consistent visual weight

Icons should appear as one unified family.

---

# AI Asset Generation

When generating icons with AI, prompts should define:

- flat vector style
- monochrome-friendly
- minimalistic
- geometric
- centered composition
- transparent background
- no text
- no shadows
- consistent stroke width

Icons should always be generated as scalable vector graphics whenever possible.

# Icon Review Checklist

Before approving a new icon:

| Question                         | Status |
| -------------------------------- | ------ |
| Recognizable within one second?  | □      |
| Consistent with existing icons?  | □      |
| Readable at small size?          | □      |
| Uses the correct color category? | □      |
| Scalable?                        | □      |
| Follows the visual language?     | □      |
| Suitable for AI regeneration?    | □      |

# Summary

The Project Genesis icon system emphasizes clarity, consistency and scalability.

Every icon should communicate meaning instantly while remaining visually unified across the entire game.

A well-designed icon system reduces cognitive load, accelerates decision-making and strengthens the overall visual identity of Project Genesis.
