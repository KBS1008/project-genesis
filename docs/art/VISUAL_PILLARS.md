# Visual Pillars

**Project:** Project Genesis  
**Version:** 1.0  
**Status:** Active  
**Last Updated:** 2026-07-12

---

# Purpose

This document defines the core visual pillars of Project Genesis.

The Visual Pillars are the highest-level artistic principles of the project. They guide every visual decision, from user interface elements to buildings, maps, icons and promotional artwork.

Whenever a visual design decision is uncertain, these pillars take precedence.

---

# Philosophy

Project Genesis is an economic simulation where information is as important as aesthetics.

The visual design should support understanding, strategic thinking and long-term gameplay.

Every asset should answer one simple question:

> Does this improve the player's understanding of the simulation?

If the answer is "no", the design should be reconsidered.

---

# Pillar 1 — Readability over Realism

## Principle

Information always has priority over visual complexity.

Players should immediately understand what they are looking at without unnecessary visual noise.

Realistic graphics are welcome, but never at the expense of clarity.

## Implications

- Clear silhouettes
- High contrast
- Simple iconography
- Limited visual clutter
- Readable at different zoom levels

## Avoid

- Photorealistic textures
- Excessive detail
- Busy backgrounds
- Visual overload

---

# Pillar 2 — Function Defines Form

## Principle

Every object should visually communicate its purpose.

A player should recognize what a building, vehicle or resource does before reading its name.

## Implications

Examples:

- A power plant should immediately communicate energy production.
- A warehouse should clearly represent storage.
- A research laboratory should appear scientific and technological.

Visual design follows functionality.

Decoration is always secondary.

---

# Pillar 3 — Industrial Elegance

## Principle

Project Genesis represents modern industry, technology and economic growth.

The world should feel efficient, organized and optimistic.

Industrial environments should appear clean and believable.

## Visual Characteristics

- Steel
- Glass
- Concrete
- Modern architecture
- Renewable energy
- Organized logistics
- Contemporary infrastructure

## Avoid

- Rust-heavy environments
- Post-apocalyptic themes
- Excessive decay
- Dark dystopian aesthetics

---

# Pillar 4 — Consistency above Variety

## Principle

Consistency is more valuable than individual artistic brilliance.

Every asset should appear as though it belongs to the same visual universe.

A coherent style strengthens immersion.

## Implications

Maintain consistency in:

- perspective
- lighting
- proportions
- colors
- materials
- iconography
- shadows
- line widths

---

# Pillar 5 — Data is Beautiful

## Principle

Data visualization is part of the game's visual identity.

Dashboards, charts, production chains and statistics are not secondary interfaces—they are central gameplay elements.

The presentation of information should feel elegant, modern and satisfying.

## Implications

- Clean charts
- Balanced spacing
- Harmonized colors
- Minimal decoration
- Professional dashboard layouts

---

# Pillar 6 — Modular by Design

## Principle

Every asset should be reusable.

Buildings, UI components, icons and overlays should form a modular design system rather than isolated artwork.

## Implications

Assets should support:

- scaling
- reuse
- recombination
- future expansion
- procedural generation where appropriate

---

# Pillar 7 — Calm Complexity

## Principle

The simulation itself is complex.

The presentation should remain calm.

The interface should help players understand complexity instead of amplifying it.

## Implications

Prefer:

- whitespace
- structured layouts
- visual hierarchy
- restrained animations
- subtle transitions

Avoid visual stress.

---

# Pillar 8 — Timeless Design

## Principle

Project Genesis should still appear modern many years after release.

Avoid short-lived design trends.

The visual identity should age gracefully.

## Prefer

- neutral colors
- simple geometry
- elegant typography
- restrained effects
- consistent spacing

Avoid fashionable effects that may quickly become outdated.

---

# Applying the Visual Pillars

Every new graphical asset should be reviewed against the following checklist.

| Question                                | Yes / No |
| --------------------------------------- | -------- |
| Is it immediately readable?             | □        |
| Does it communicate its function?       | □        |
| Does it match the industrial style?     | □        |
| Does it fit existing assets?            | □        |
| Does it improve information clarity?    | □        |
| Is it modular and reusable?             | □        |
| Does it avoid unnecessary decoration?   | □        |
| Will it still look modern in ten years? | □        |

Only assets that satisfy all pillars should become part of the official asset library.

---

# Priority Order

If two pillars conflict, the following priority applies:

1. Readability over Realism
2. Function Defines Form
3. Consistency above Variety
4. Data is Beautiful
5. Industrial Elegance
6. Calm Complexity
7. Modular by Design
8. Timeless Design

# Summary

The Visual Pillars define the immutable artistic foundation of Project Genesis.

Every visual decision should reinforce clarity, consistency, functionality and long-term maintainability.

The goal is not to create the most spectacular graphics, but the most understandable and enduring visual language for a modern economic simulation.
