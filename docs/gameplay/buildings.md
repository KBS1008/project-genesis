# Buildings

> Beschreibt das Gebäudesystem von Project Genesis.

Version: 1.0

---

# Zweck

Gebäude bilden die physische Infrastruktur eines Unternehmens.

Sie ermöglichen:

- Produktion
- Lagerung
- Energieversorgung
- Forschung
- Verwaltung
- Logistik

Gebäude sind die Grundlage jeder wirtschaftlichen Aktivität.

---

# Designziele

Das Gebäudesystem soll:

- leicht verständlich sein
- strategische Entscheidungen fördern
- langfristige Planung belohnen
- viele Spezialisierungen ermöglichen
- einfach erweiterbar sein

---

# Grundprinzip

```
Grundstück

↓

Gebäude errichten

↓

Produktion ermöglichen

↓

Unternehmen erweitern

↓

Neue Technologien freischalten
```

---

# Gebäude besitzen

Jedes Gebäude besitzt:

- einen Gebäudetyp
- eine Position
- einen Besitzer
- einen Status
- eine Ausbaustufe
- eine Effizienz
- Energiebedarf oder Energieerzeugung
- Wartungsbedarf
- Produktionskapazität

---

# Gebäudekategorien

## Produktion

Produziert Ressourcen oder Produkte.

Beispiele

- Sägewerk
- Mine
- Stahlwerk
- Farm

---

## Lager

Lagert Ressourcen.

Beispiele

- Lagerhaus
- Silo
- Tanklager

---

## Energie

Versorgt andere Gebäude.

Beispiele

- Kohlekraftwerk
- Solarkraftwerk
- Umspannwerk

---

## Forschung

Schaltet neue Technologien frei.

Beispiele

- Forschungslabor
- Entwicklungszentrum

---

## Verwaltung

Verbessert Unternehmensfunktionen.

Beispiele

- Hauptquartier
- Verwaltungsgebäude

---

## Infrastruktur

Verbessert Versorgung.

Beispiele

- Straßen
- Schienen
- Verteilzentren

---

# Bau eines Gebäudes

Der Bau erfolgt in mehreren Schritten.

```
Bauauftrag

↓

Ressourcen reservieren

↓

Bauzeit

↓

Fertigstellung

↓

Gebäude aktiv
```

Während der Bauzeit steht das Gebäude nicht zur Verfügung.


# Ausbau

Fast alle Gebäude besitzen mehrere Ausbaustufen.

Ein Ausbau kann verbessern:

- Produktionsgeschwindigkeit
- Energieverbrauch
- Lagerkapazität
- Produktionslinien
- Wartungskosten

Version 1

Maximalstufe:

```
10
```

---

# Module

Später können Gebäude erweitert werden.

Beispiele

- Geschwindigkeitsmodul
- Energiesparmodul
- Qualitätsmodul
- KI-Steuerung

Version 1

Module sind vorbereitet, aber noch nicht verfügbar.

---

# Wartung

Alle Gebäude verschleißen.

Je länger ein Gebäude arbeitet,

desto höher wird der Wartungsbedarf.

Schlechte Wartung reduziert:

- Effizienz
- Produktionsgeschwindigkeit
- Zuverlässigkeit

---

# Energie

Produktionsgebäude benötigen Energie.

Fehlt Energie,

kann keine Produktion stattfinden.

Später können Prioritäten vergeben werden.

---

# Produktion

Ein Gebäude produziert niemals selbst.

Es führt lediglich Rezepte aus.

Dadurch kann dasselbe Gebäude später neue Produkte herstellen.

---

# Forschung

Neue Technologien können:

- Gebäude verbessern
- neue Gebäude freischalten
- Module erlauben
- Effizienz steigern

---

# Grundstück

Alle Gebäude befinden sich auf dem Unternehmensgelände.

Version 1

Unbegrenzte Baufläche.

Version 2

Erweiterbares Grundstück.

---

# Balance

Gebäude sollen niemals "fertig" sein.

Der Spieler entscheidet ständig:

- Neues Gebäude?
- Ausbau?
- Wartung?
- Spezialisierung?
- Energie sparen?
- Produktion erhöhen?

Dadurch bleibt das Spiel langfristig interessant.
